# AURA Faucet Governance

## Roles

**Maintainers**: Core team members with merge access
**Contributors**: Community members who submit PRs

## Decision Making

| Change Type | Approval Required |
|-------------|-------------------|
| Bug fixes | 1 maintainer |
| New features | 2 maintainers |
| Breaking changes | Core team consensus |

## Release Process

1. Changes merged to `main`
2. Version tagged following semver
3. Release notes published
4. Docker image built and pushed

## Rate Limits

Faucet rate limits are set by the core team based on testnet needs. Community can propose changes via GitHub Issues.
